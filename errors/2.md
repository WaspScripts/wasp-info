The error message looks something like this:
```
Succesfully compiled in 1440.62 milliseconds.
[00:00:01:0747]:[WLAssets]: Updating ...
[00:00:09:0796]:[WLAssets]: Downloading files. 1 files left, please wait.
[00:00:09:0796]:[WLAssets]: Files left: [...]
[00:00:17:0813]:[WLAssets]: Downloading files. 1 files left, please wait.
[00:00:17:0813]:[WLAssets]: Files left: [...]
Runtime error: "[00:00:21:0640]:[WLAssets]: Timed out downloading assets: [...]" at line 142, column 9 in file "Includes/WaspLib/utils/assets.simba"
Stack Trace:
  0) Line 142 in function "TWLAssets.Update" in file "Includes/WaspLib/utils/assets.simba"
  1) Line 165 in function "WaspLibSetup" in file "Includes/WaspLib/main/interfaces/setup.simba"
  2) Line 238 in "main" in file "Includes/WaspLib/main/interfaces/setup.simba"
The following objects were not freed:
  TSimbaHTTPClient (000000000C3B05B0)
[00:00:21:0641]:[WLAssets]: Access violation
```

This can mean a couple of things:
- The speed of the internet connection between you and WaspScripts server is slower than expected
- You have no internet
- You have been spamming WaspScripts servers, intentionally or not and sre temporarily rate limited, this will go away after 1h
- Something on your end is blocking you from accessing my server (firewall, ISP, etc)
- My servers are down (and you will have to wait for me to fix it)

Usually, trying again after some time has passed will solve the issue, however, if you are in an hurry, you can try to manually download the assets and place them in the right places:
[wasp-assets](https://github.com/WaspScripts/wasp-assets)
